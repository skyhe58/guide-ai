## Q1: YOLO 系列的架构演进？YOLOv8 相比之前版本有哪些改进？

**难度**：⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：版本演进脉络 → YOLOv8 关键改进 → 性能对比

**标准答案**：YOLO 从 v1 到 v8 经历了多次重大改进。YOLOv1 开创单阶段检测；v3 引入多尺度检测和 FPN；v5 用 PyTorch 重写，工程化优秀；v8 是 Ultralytics 最新版本，关键改进包括：(1) Anchor-Free 设计，不再需要预定义 Anchor Box；(2) C2f 模块替代 C3，更多梯度流分支；(3) Decoupled Head，分类和回归解耦；(4) Task-Aligned Assigner 动态标签分配；(5) 统一框架支持检测/分割/姿态估计/分类多任务。

**追问**：
- Anchor-Based 和 Anchor-Free 的区别？（预定义框 vs 直接预测中心点偏移）
- YOLO 的 Backbone-Neck-Head 各自作用？（特征提取-特征融合-预测输出）
- NMS 的替代方案？（Soft-NMS、DIoU-NMS、端到端检测器 DETR）

## Q2: mAP 是如何计算的？mAP50 和 mAP50-95 有什么区别？

**难度**：⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：IoU → Precision/Recall → AP → mAP → 两种标准

**标准答案**：首先计算 IoU（预测框与真实框的交并比），根据 IoU 阈值判定 TP/FP。将所有预测按置信度排序，逐个计算 Precision 和 Recall，绘制 P-R 曲线，曲线下面积即为 AP（单类）。mAP 是所有类别 AP 的平均值。mAP50 在 IoU=0.5 阈值下计算（PASCAL VOC 标准），评估较宽松。mAP50-95 在 IoU=0.5 到 0.95（步长 0.05）共 10 个阈值下分别计算再取平均（COCO 标准），更全面严格，能区分定位精度差异。

**追问**：
- Precision 和 Recall 的权衡？什么场景侧重哪个？（安全场景重 Recall，精确场景重 Precision）
- IoU 的变体有哪些？（GIoU、DIoU、CIoU，解决不重叠时梯度为 0 的问题）
- 如何处理类别不平衡对评估的影响？（mAP 本身对每类平等加权，但可以加权 mAP）

## Q3: Diffusion Model 的核心原理是什么？与 GAN 相比有什么优劣？

**难度**：⭐⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：前向/反向过程 → 训练目标 → 与 GAN 对比

**标准答案**：Diffusion Model 包含前向和反向两个过程。前向过程逐步向数据添加高斯噪声（T 步后变成纯噪声），反向过程训练 UNet 预测每步噪声，从而逐步去噪恢复数据。训练目标是最小化预测噪声与真实噪声的 MSE。与 GAN 对比：Diffusion 优势——训练稳定（无模式崩塌）、生成质量高、可控性强（CFG）；劣势——生成速度慢（需要多步迭代），但 DDIM 等加速方法已将步数降到 20-50。

**追问**：
- DDPM 和 DDIM 的区别？（DDIM 确定性采样，可跳步加速 20-50 倍）
- Classifier-Free Guidance 的原理？（无条件和有条件预测的加权组合，w 控制引导强度）
- 噪声调度（Noise Schedule）的作用？（控制每步加噪量，余弦调度优于线性）

## Q4: Stable Diffusion 的架构是什么？为什么在潜空间做扩散？

**难度**：⭐⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：三大组件 → 潜空间优势 → 生成流程

**标准答案**：SD 由三个核心组件组成：(1) VAE 将 512×512 图像压缩到 64×64 潜空间（压缩 48 倍）；(2) UNet 在潜空间进行条件去噪，通过 Cross-Attention 融合文本条件；(3) CLIP Text Encoder 将文本编码为条件向量。在潜空间做扩散的优势：计算量降低 48 倍，使消费级 GPU（8GB 显存）可运行；潜空间语义更紧凑，去除了像素级冗余信息。

**追问**：
- Cross-Attention 在 UNet 中如何工作？（Q 来自图像特征，K/V 来自文本嵌入）
- ControlNet 的原理？（复制 UNet Encoder，添加条件输入，零卷积连接）
- SD 版本演进？（1.5→SDXL→SD3，分辨率和架构持续改进）

## Q5: 多模态模型（VLM）的架构有哪些模式？LLaVA 的训练策略是什么？

**难度**：⭐⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：三种架构模式 → LLaVA 具体架构 → 两阶段训练

**标准答案**：VLM 三种主要架构：(1) Cross-Attention 模式（Flamingo）——在 LLM 层间插入交叉注意力层；(2) 投影拼接模式（LLaVA）——Projector 将视觉 token 映射后与文本 token 拼接输入 LLM；(3) 原生多模态（Gemini）——统一 Transformer 处理所有模态。LLaVA 训练分两阶段：阶段 1 冻结 Vision Encoder 和 LLM，只训练 Projector 对齐特征空间（595K 图文对）；阶段 2 冻结 Vision Encoder，训练 Projector 和 LLM（158K 视觉指令数据）。

**追问**：
- 为什么用 CLIP 作为 Vision Encoder？（已学习视觉-语言对齐）
- GPT-4V、Gemini、Qwen-VL 各有什么特点？（GPT-4V 最强，Qwen-VL 中文最好且开源）
- 如何评估 VLM 的能力？（MMBench、MMMU、TextVQA 等基准）

## Q6: 语义分割、实例分割和全景分割的区别？

**难度**：⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：三者定义 → 关键区别 → 代表模型 → 应用场景

**标准答案**：语义分割为每个像素分配类别标签，不区分同类不同实例。实例分割在检测基础上为每个实例生成独立掩码，区分同类不同实例但不处理背景。全景分割结合两者，前景做实例分割，背景做语义分割。代表模型：语义分割——DeepLabV3+；实例分割——Mask R-CNN、YOLO-Seg；全景分割——Panoptic FPN。评估指标：语义分割用 mIoU，实例分割用 mask AP。

**追问**：
- Mask R-CNN 的 RoI Align 相比 RoI Pooling 有什么改进？（双线性插值避免量化误差）
- YOLO-Seg 和 Mask R-CNN 如何选择？（YOLO-Seg 快，Mask R-CNN 精度高）
- mIoU 和 Pixel Accuracy 哪个更可靠？（mIoU，不受类别不平衡影响）

## Q7: SAM（Segment Anything）的架构和 Prompt 机制是什么？

**难度**：⭐⭐⭐ | **频率**：🔥🔥🔥

**答题思路**：三大组件 → Prompt 类型 → 零样本能力 → 应用

**标准答案**：SAM 由三部分组成：(1) Image Encoder（ViT-H）提取图像特征，重量级但只需运行一次；(2) Prompt Encoder 编码用户提示（点、框、掩码、文本）；(3) Mask Decoder（轻量 Transformer）融合特征生成掩码。Prompt 机制让 SAM 实现零样本分割——通过点击、框选等方式指定目标，无需针对特定类别训练。在 11M 图像、1B 掩码上训练，泛化能力极强。

**追问**：
- SAM 和传统分割模型的本质区别？（Prompt 驱动 vs 类别驱动）
- SAM 2 相比 SAM 1 有什么改进？（支持视频、Memory 机制、6x 更快）
- SAM + YOLO 如何组合使用？（YOLO 检测框作为 SAM 的 Box Prompt）

## Q8: 迁移学习的原理是什么？CV 模型微调有哪些策略？

**难度**：⭐⭐⭐ | **频率**：🔥🔥

**答题思路**：原理 → 为什么有效 → 微调策略 → 实践建议

**标准答案**：迁移学习利用在大规模数据集上预训练的权重作为初始化。有效原因：CNN 浅层学习的边缘、纹理等低级特征具有通用性；预训练提供好的参数初始化，加速收敛并减少过拟合。微调策略：(1) 只训练检测头（数据极少 <200 张）；(2) 冻结 Backbone 训练 Neck+Head（数据较少 200-1000 张）；(3) 全参数微调（数据充足 >1000 张）。关键：微调学习率要比从头训练低 10 倍，防止破坏预训练特征。

**追问**：
- 灾难性遗忘如何避免？（低学习率、冻结层、EWC 正则化）
- 源域和目标域差异很大时怎么办？（两阶段迁移，先在相似中间域训练）
- 小样本训练有哪些策略？（强数据增强、冻结更多层、正则化）

## Q9: 模型导出有哪些格式？ONNX 和 TensorRT 的区别？

**难度**：⭐⭐⭐ | **频率**：🔥🔥

**答题思路**：主要格式 → ONNX 定位 → TensorRT 优势 → 选型建议

**标准答案**：主要导出格式：ONNX（通用中间格式）、TensorRT（NVIDIA GPU 优化）、CoreML（Apple 设备）、OpenVINO（Intel CPU）、TFLite（移动端）。ONNX 是框架无关的中间格式，兼容性最好，可作为转换桥梁。TensorRT 针对 NVIDIA GPU 深度优化，支持 FP16/INT8 量化，推理速度比 ONNX Runtime 快 2-3 倍。选型：通用部署用 ONNX，NVIDIA GPU 生产环境用 TensorRT，Apple 设备用 CoreML。

**追问**：
- FP16 和 INT8 量化的区别？（FP16 精度损失极小速度 2x，INT8 需要校准数据速度 3-4x）
- 动态 shape 和静态 shape 的区别？（动态灵活但性能低，静态性能高但不灵活）
- 导出后如何验证精度？（对比原始模型和导出模型的 mAP 差异，应 <0.5%）

## Q10: Canny 边缘检测的步骤是什么？

**难度**：⭐⭐ | **频率**：🔥🔥

**答题思路**：五个步骤逐一说明 → 每步作用 → 参数选择

**标准答案**：Canny 包含五个步骤：(1) 高斯滤波平滑降噪；(2) Sobel 算子计算水平和垂直方向梯度；(3) 非极大值抑制（NMS），沿梯度方向只保留局部最大值，使边缘变细；(4) 双阈值检测，将像素分为强边缘、弱边缘和非边缘；(5) 边缘连接，与强边缘相连的弱边缘保留，否则丢弃。双阈值比例通常 1:2 或 1:3。

**追问**：
- NMS 在边缘检测和目标检测中的区别？（边缘检测沿梯度方向抑制，目标检测按 IoU 抑制）
- 如何自动选择 Canny 阈值？（Otsu 方法或中值法）

## 面试准备建议

1. **YOLO 是重中之重**：架构演进、训练流程、评估指标必须熟练
2. **Diffusion 原理必考**：前向/反向过程、DDPM/DDIM、CFG 要能清晰解释
3. **多模态是趋势**：LLaVA 架构、VLM 对比是新热点
4. **动手实践**：至少跑通一个 YOLO 训练 + 导出 + 部署的完整流程
5. **关注最新进展**：SAM 2、SDXL、GPT-4o 等最新模型要了解
